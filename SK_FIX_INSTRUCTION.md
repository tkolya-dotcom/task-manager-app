# Исправление: Не отображается информация о СК в монтаже

## Проблема
При открытии вкладки "Подробно" в созданном монтаже не отображается:
- ID СК
- Наименование (марка/модель)  
- Статус оборудования
- Тип по договору
- Плановая дата
- Информация о заявке на материалы

## Причина
В базе данных в таблице `installations` отсутствовали колонки для хранения информации о кондиционерах (СК).

## Решение

### ✅ Шаг 1: Выполните SQL скрипт в Supabase

**Файл:** `c:\Users\Tkolya\Desktop\мои\планировщик\sql\add_sk_fields_to_installations.sql`

**Как выполнить:**
1. Откройте https://supabase.com/dashboard/project/YOUR_PROJECT/sql
2. Скопируйте содержимое файла `add_sk_fields_to_installations.sql`
3. Вставьте в SQL Editor
4. Нажмите **Run**
5. Убедитесь, что выполнено успешно

**Что добавляется:**
- Поля: `id_ploshadki`, `servisnyy_id`, `rayon`, `planovaya_data_1_kv_2026`
- Поля СК 1-6: `id_sk1`..`id_sk6`, `naimenovanie_sk1`..`naimenovanie_sk6`, `status_oborudovaniya1`..`status_oborudovaniya6`, `tip_sk_po_dogovoru1`..`tip_sk_po_dogovoru6`
- Индексы для ускорения поиска

### ✅ Шаг 2: Проверка успешности

Выполните SQL запрос для проверки:

```sql
-- Проверить наличие столбцов
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'installations' 
AND column_name LIKE '%sk%'
ORDER BY column_name;
```

Должны отобразиться все добавленные поля.

### ✅ Шаг 3: Обновите существующие монтажа (если нужно)

Если у вас уже есть созданные монтажи, данные в поля СК можно добавить через редактирование:

1. Откройте страницу редактирования монтажа
2. Заполните поля СК
3. Сохраните

Или выполните SQL для обновления конкретных записей:

```sql
UPDATE installations 
SET 
  id_sk1 = 'значение',
  naimenovanie_sk1 = 'значение',
  tip_sk_po_dogovoru1 = 'значение'

```

## Проверка через консоль браузера

При открытии вкладки "Подробно" откройте консоль (F12) и убедитесь, что видите логи:

```

```

В объекте `All installation fields` должны быть видны значения полей:
- `id_sk1`, `naimenovanie_sk1`, `status_oborudovaniya1`, `tip_sk_po_dogovoru1`
- `planovaya_data_1_kv_2026`
- И другие поля СК
