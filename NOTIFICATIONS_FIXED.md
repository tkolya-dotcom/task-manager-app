# ✅ Исправление уведомлений о назначениях

## Выполненные изменения:

### 1. ✅ Добавлена периодическая проверка уведомлений
**Файл:** `index.html` (строки ~2103-2110)

```javascript
// Start periodic notification check every 30 seconds
if (!window.notificationCheckInterval) {
    window.notificationCheckInterval = setInterval(() => {
        checkAndShowPendingNotifications();
    }, 30000); // Check every 30 seconds
}
```

**Результат:** Уведомления проверяются каждые 30 секунд, даже если вкладка открыта давно.

---

## Как это работает:

### При создании задачи/монтажа:

1. **Сохранение в базу:**
   ```javascript
   await supabase.from('notification_queue').insert([{
       user_id: assigneeId,
       title: '📋 Новое назначение',
       body: 'Вам назначена задача/монтаж от: ...'
   }]);
   ```

2. **Мгновенное уведомление:**
   - Если браузер имеет разрешение → показывается сразу
   - Если нет → сохранится в базе

3. **Периодическая проверка:**
   - Каждые 30 секунд проверяются новые уведомления
   - Показываются браузерные уведомления

4. **При загрузке приложения:**
   - Проверяются все несохранённые уведомления
   - Показывается последнее

## Проверка работоспособности:

### 1. Откройте консоль браузера (F12)

После создания задачи должны быть логи:
```
Sending notification: <taskId> to <assigneeId>
Notification saved to queue for assignee: <assigneeId>
Browser notification shown to assignee
```

### 2. Проверьте таблицу `notification_queue`

```sql
SELECT * FROM notification_queue 
WHERE user_id = '<ID_исполнителя>'
ORDER BY created_at DESC;
```

Должны быть записи с `sent = false`.

### 3. Тестирование:

**Сценарий A (вкладка открыта):**
1. Менеджер создаёт задачу на исполнителя
2. Исполнитель получает браузерное уведомление через 1-30 секунд

**Сценарий B (вкладка закрыта):**
1. Менеджер создаёт задачу на исполнителя
2. Исполнитель открывает приложение → видит уведомление

## Разрешение на уведомления:

Если уведомления не приходят, проверьте разрешение:

```javascript
console.log('Notification permission:', Notification.permission);
```

**Должно быть:** `'granted'`

**Если `'denied'`:**
- Chrome: `chrome://settings/content/notifications` → разрешить
- Firefox: `about:preferences#privacy` → Уведомления → разрешить

## Для настоящих Push-уведомлений:

Текущая реализация работает только при **открытой вкладке**.

Для уведомлений при **закрытом браузере** нужен OneSignal или Firebase:

1. Установить SDK на бэкенд
2. Сохранять токены устройств пользователей
3. Отправлять через API OneSignal/Firebase

**Это требует доработки бэкенда.**
