# Диагностика: Не отображается информация о СК в монтаже

## Проблема
При открытии вкладки "Подробно" в созданном монтаже не отображается информация о кондиционере:
- ID СК
- Наименование
- Статус
- Тип по договору
- Плановая дата
- Создана ли заявка на материалы

## Решение по шагам

### Шаг 1: Проверка базы данных

Выполните SQL скрипт в Supabase SQL Editor:

```sql
-- Путь к файлу: c:\Users\Tkolya\Desktop\мои\планировщик\sql\add_sk_fields_to_installations.sql
```

**Как выполнить:**
1. Откройте https://supabase.com/dashboard/project/YOUR_PROJECT/sql
2. Скопируйте содержимое файла `add_sk_fields_to_installations.sql`
3. Нажмите "Run"
4. Убедитесь, что столбцы добавлены в таблицу `installations`

**Проверка:**
```sql
-- Проверить наличие столбцов
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'installations' 
AND column_name LIKE '%sk%'
ORDER BY column_name;
```

### Шаг 2: Проверка данных в существующих монтажах

Проверьте, есть ли данные в полях СК:

```sql
SELECT 
  id, 
  title,
  id_sk1,
  naimenovanie_sk1,
  tip_sk_po_dogovoru1,
  planovaya_data_1_kv_2026
FROM installations
LIMIT 10;
```

Если поля пустые (NULL), значит данные не были сохранены при создании монтажа.

Backend уже настроен правильно (файл `backend/src/routes/installations.js`):
- Строки 172-198: GET /:id возвращает все поля через `.select('*')`
- Строки 206-282: POST создает монтаж со всеми полями СК
- Строки 285-384: PUT обновляет монтаж со всеми полями СК

**Backend работает корректно, изменения не требуются!**

2. **Проверьте через консоль браузера:**

3. **Проверьте отображение:**

     - Информация о СК (ID, Наименование, Статус, Тип по договору)
     - Статус заявки на материалы

### Шаг 6: Если данные всё ещё не отображаются

**Проверьте журнал консоли браузера:**

Откройте консоль (F12) и найдите логи:
```
Installation data received: {...}
All installation fields: {...}
```

Если видите `null` или `undefined` для полей СК:
1. ✅ Backend возвращает данные → проблема в базе данных
2. ❌ Backend не возвращает данные → проверьте SQL запрос (строка 174)

**Проверьте запрос в базе данных:**
```sql
-- Выполните вручную с реальным ID монтажа
SELECT * FROM installations WHERE id = 'YOUR_INSTALLATION_ID';
```

Убедитесь, что поля `id_sk1`, `naimenovanie_sk1`, `tip_sk_po_dogovoru1` и другие содержат данные.

## Возможные проблемы и решения

### Проблема 1: SQL ошибка при добавлении столбцов
**Решение:** Столбцы уже существуют. Это нормально, используется `IF NOT EXISTS`.

### Проблема 2: Данные не сохраняются при создании
**Решение:** Проверьте, что frontend отправляет все поля. См. файл `frontend/src/pages/Installations.jsx` (создание монтажа).

### Проблема 3: RLS политики блокируют доступ
**Решение:** Проверьте политики доступа в Supabase Dashboard → Authentication → Policies.

### Проблема 4: Кэш браузера
**Решение:** Очистите кэш браузера (Ctrl+Shift+Delete) или откройте в режиме инкогнито.
