## 🚀 ЧТО НУЖНО СДЕЛАТЬ ВАМ

### Шаг 1: Добавьте недостающие поля (если нет)

Выполните SQL запрос в Supabase:

```sql
-- Проверяем наличие servisnyy_id
SELECT column_name 
FROM information_schema.columns 
WHERE table_name = 'installations' 
AND column_name = 'servisnyy_id';

-- Если отсутствует, добавляем:
ALTER TABLE installations
ADD COLUMN IF NOT EXISTS servisnyy_id TEXT;
```

### Шаг 2: Проверьте данные в базе

Выполните SQL запрос для проверки конкретного монтажа:

```sql
SELECT 
  id,
  title,
  id_sk1,
  naimenovanie_sk1,
  status_oborudovaniya1,
  tip_sk_po_dogovoru1,
  planovaya_data_1_kv_2026
FROM installations 
WHERE id = '32688418-2061-4246-b8ff-520847298176'; -- ваш ID монтажа
```

**Если все поля NULL:**
- Данные ещё не записаны в базу
- Нужно создать новый монтаж или обновить существующий через интерфейс

**Если есть значения:**
- ✅ Данные в базе есть
- Переходите к шагу 3

## 🔧 Возможные проблемы и решения

### Проблема 1: Все поля NULL в логах

**Решение:**
1. Убедитесь, что SQL скрипт выполнен
2. Создайте новый монтаж с заполненными полями СК
3. Или обновите существующий через редактирование

### Проблема 2: Ошибка "Cannot read property"

**Решение:**
Очистите кэш браузера (Ctrl+Shift+Delete) и перезагрузите страницу (Ctrl+F5)

### Проблема 3: Плановая дата не отображается

**Причина:** Разный формат даты (Unix timestamp vs ISO string)

**Решение:** Уже исправлено в строке 3143 - теперь поддерживает оба формата

### Проблема 4: Нет поля servisnyy_id

**Решение:** Добавить через SQL (см. Шаг 1 выше)
