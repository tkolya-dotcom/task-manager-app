# Настройка коротких ID для сущностей

## Что сделано

Добавлена сквозная нумерация для всех сущностей в формате 4-значного кода:
- Задачи (tasks) → 0001, 0002, ...
- Монтажи (installations) → 0001, 0002, ...
- Заявки на материалы (purchase_requests) → 0001, 0002, ...
- Проекты (projects) → 0001, 0002, ...

## Инструкция по установке

### 1. Выполните SQL миграцию в Supabase

```sql
-- Скопируйте и выполните весь файл sql/add_short_id.sql в SQL Editor в Supabase
```

Файл миграции: `sql/add_short_id.sql`

**Что сделает миграция:**
- ✅ Создаст таблицу счетчиков `id_counters`
- ✅ Добавит функцию `get_next_id()` для автоинкремента
- ✅ Добавит поле `short_id` во все таблицы
- ✅ Создаст триггеры для автоматического присвоения номера
- ✅ Создаст индексы для ускорения поиска

### 2. Проверьте работу

После выполнения миграции:

1. **Обновите страницу** (Ctrl+F5)
2. **Создайте новую задачу/монтаж/заявку** - должен присвоиться номер `0001`
3. **Проверьте карточку** - в заголовке появится бейдж с номером
4. **Откройте вкладку "Подробно"** - внизу отображается полный ID

### 3. Поиск по номеру

Во вкладке **"Монтажи"** появилось поле поиска:
- Введите `0001` → найдет монтаж с short_id=1
- Введите `0025` → найдет монтаж с short_id=25

## Технические детали

### Формат отображения
- В списке: `#0001 Название задачи` (бейдж синего цвета)
- В деталях: `ID: 0001` вместо UUID
- При поиске: точное совпадение числа

### Логика работы
```javascript
// Отображение с leading zeros
String(inst.short_id).padStart(4, '0')  // 1 → "0001"

// Поиск
parseInt("0001", 10)  // "0001" → 1
```

### Структура БД
```sql
id_counters:
- entity_type: 'tasks' | 'installations' | 'purchase_requests' | 'projects'
- current_value: INTEGER (текущий номер)
- updated_at: TIMESTAMPTZ

Триггер при INSERT автоматически устанавливает short_id
```

## Решение проблем

### Если номера не присваиваются
1. Проверьте выполнение миграции в Supabase
2. Убедитесь что триггеры созданы: 
   ```sql
   SELECT * FROM pg_trigger WHERE tgname LIKE '%set_short_id%';
   ```

### Если поиск не работает
1. Откройте консоль браузера (F12)
2. Проверьте нет ли ошибок JavaScript
3. Убедитесь что поле `installationSearch` существует в DOM

### Сброс счетчиков (если нужно)
```sql
UPDATE id_counters SET current_value = 0;
```

⚠️ **Внимание!** Сброс удалит все текущие номера!
